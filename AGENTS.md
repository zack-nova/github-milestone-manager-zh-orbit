# GHPM Orbit

用于通过 `gh-issue-sync` 管理 GitHub issue、milestone、开发准备和 PR 提交。

先读所需子文档：长期规则在 `docs/ghpm/`。

相关 skill：`$create-issue`、`$issue-triage`、`$issue-evaluation`、`$create-milestone`、`$prepare-development`、`$submit-pr`。

硬规则：每个 issue 只保留一个 `status/*`；不要提交 `.issues/` 或 `.ghpm.json`；批量 GitHub、worktree、PR 修改前先 dry-run。
